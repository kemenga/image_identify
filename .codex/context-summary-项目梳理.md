## 项目上下文摘要（项目梳理）

生成时间：2026-04-20 09:55:18 CST

### 1. 相似实现分析

- **实现1**: `main.py`
  - 模式：轻量 CLI 入口
  - 可复用：参数解析、结果打印、报告落盘流程
  - 需注意：入口非常薄，核心逻辑都在下层模块
- **实现2**: `detector.py`
  - 模式：上层统一封装 + 多证据融合
  - 可复用：`UniversalTamperDetector`、`build_report`、`visualize_detection`
  - 需注意：对外输出协议基本由这里定义
- **实现3**: `document_detector.py`
  - 模式：大文件集中式规则引擎
  - 可复用：候选生成、候选筛选、局部特征、区域异常检测
  - 需注意：当前文件职责较多，阅读成本最高
- **实现4**: `preprocessing.py`
  - 模式：基础图像处理独立下沉
  - 可复用：`load_image`、`detect_text_lines`、`segment_line_characters`
  - 需注意：上层默认消费这里的文本行和字符上下文

### 2. 项目约定

- **命名约定**: Python 使用 `snake_case`，数据模型使用 `@dataclass(slots=True)`
- **文件组织**: 按“入口 / 融合 / 规则 / 预处理 / 测试”分层
- **导入顺序**: 标准库、第三方库、本地模块
- **代码风格**: 中文注释说明意图，4 空格缩进

### 3. 可复用组件清单

- `main.py`：CLI 参数与结果打印
- `detector.py`：统一结果模型、报告结构、可视化
- `document_detector.py`：文档规则候选生成
- `preprocessing.py`：文本行和字符切分
- `test_detector.py`：样例级回归验证

### 4. 测试策略

- **测试框架**: `unittest`
- **测试模式**: 三类样例图回归 + CLI 冒烟 + 异常场景
- **参考文件**: `test_detector.py`
- **覆盖要求**: 样例命中、输出文件存在、异常路径合理

### 5. 依赖和集成点

- **外部依赖**: `opencv-python`、`numpy`、`Pillow`
- **内部依赖**: `main.py -> detector.py -> document_detector.py / preprocessing.py`
- **集成方式**: 本地函数调用
- **配置来源**: `requirements.txt`

### 6. 技术选型理由

- **为什么用这个方案**: 当前仓库针对文档型图像，用传统视觉规则能更快起量，不依赖模型权重
- **优势**: 轻量、本地可跑、可解释性较强
- **劣势和风险**: 规则集中在一个大文件，扩展成本较高，对自然场景泛化有限

### 7. 关键风险点

- **结构风险**: `document_detector.py` 模块过大，后续维护和拆分成本会上升
- **边界条件**: 文本上下文少、模糊、遮挡时，更多依赖兜底区域异常检测
- **验证风险**: 当前测试依赖样例图，覆盖面主要集中在 3 类文档场景
